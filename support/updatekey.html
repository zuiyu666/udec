Update Key

When users purchase a program upgrade, pre-purchase the next version of the program or add a new option to an existing license, most often the security key will be updated using a key reconfiguration process that is performed directly in UDEC. Any single change to the current license is performed using a password file; multiple changes to the license at the same time are made with multiple password files. The steps are as follows.

  1. Itasca sends a password file(s) for the change(s) to be made to the license. Have the file(s) in hand.
  2. Ensure the key to be updated is attached to the computer.
  3. Start UDEC.
  4. Go to the Update Key dialog; follow the menu sequence Tools —> Update Key….

Figure 1: The Update Key dialog. Click to enlarge in a new window.


  1. Use the dialog controls to select the password file(s) needed to update the key. The process can be completed by performing these steps on each password file in hand one at a time; however, it is easier and faster to specify all password files at the same time (hold the Ctrl key while clicking on each file needed, which will require that all files be contained in the same folder).
  2. Successful completion of the process is indicated in a message dialog. Unsuccessful completion will terminate in an error. In the case of an error, please contact the code-selling office or agent that supplied your password file(s) immediately to resolve the issue.
